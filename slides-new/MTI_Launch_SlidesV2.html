Innovative approaches to capture, analyze, visualize human mobility patterns

Equipped with state-of-art big geospatial data computing platforms, researchers from the CGIS conduct research in

  • human mobility and spatial trajectory analytics
  • place-based analytics
  • 3D geovisualization
  • geospatial semantic modeling

We develop innovative approaches to retrieve, model and visualize human mobility patterns from diverse data sources (e.g., large scale vehicle GPS trajectories, location-based social media, and human activity logs)

Using big GPS trajectory datasets, we estimate VMT for Maryland roads of different function classes

INRIX trajectory data
  • Collected using GPS trackers (embedded GPS)and mobile devices (using an app)
  • GPS waypoints with unique ID on each trip

    • Four months of data from 2015
  • February (3,565,361 trips, 398,652,043 waypoints);
  • June (4,835,882 trips, 152,604,037 waypoints);
  • July (4,868,583 trips, 100,246,683 waypoints);
  • October (6,420,576 trips, 725,217,440 waypoints)

    • Uncertainty in GPS positioning, simply using nearest segment to snap the waypoint is problematic

  • Find the 10 nearest neighbor as the candidates.

  • Select the candidate with largest cosine

  • Example: a1 < a2, but segment 2 is the best candidate

    		•

    Large and irregular time intervals between two waypoints (1~2mins) may cause gaps between road segments in the same trip.

  • Assuming drivers take the shortest path between the two recorded waypoints
    •
  • Shortest paths generated by Dijkstra algorithm are used to fill the gaps

    • Using Spark for trajectory reconstruction

  • “Points” blocks are split into subsets from the original data set.

  • HERE road network is copied to each pipeline as broadcast variable (memory intensive)
  • Spatial computing is done by a Python UDF on Spark
